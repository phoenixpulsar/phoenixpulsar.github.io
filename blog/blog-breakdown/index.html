Blog Breakdown: Unifying My Disjointed Blogs

July 14, 2026 · Phoenix Pulsar · 3 min read

My writing had quietly sprawled across two completely different engines, and I only noticed when I tried to send someone "all my blogs" and realized I couldn't. This post is the breakdown of what was scattered, and how I pulled it back together.

The problem: two engines, one site

This site is a hybrid. The homepage is a hand-authored React "Signal Hub". The tech-journal posts — Welcome, Do things live in the blockchain forever?, and How to check ownership of an NFT in an Auction? — are built with Docusaurus and live under /blog. Then there are standalone "field-note" blogs, Spirit in the Sky, Emotional Debt, and Why the App Layer Isn't Dead, that were once tucked away inside the docs sidebar and now live full-screen under /notes.

Nothing linked these worlds together. The hub only pointed at "Notes" and "Blog"; the note-blogs were reachable only by digging through a documentation sidebar; and there was no single place that listed everything.

What changed

Five things, all aimed at discoverability:

  1. Hub navigation — the Signal Hub now surfaces the note-blogs directly, and its "Blog" node points at a new unified index instead of just the Docusaurus feed.
  2. A unified index — /all-blogs/ lists every blog regardless of engine, each with a short description and a direct link. One place, everything.
  3. Standalone note-blogs — the field notes shed the documentation sidebar and now read as their own blog entries, with tabs that link back to the index.
  4. Shared cross-navigation — a consistent bar at the foot of every blog (this one included) links to the hub, the unified index, and each individual blog, so you can move between engines without hitting a dead end.
  5. This post — documenting the reorganization so the structure is written down, not just implied.

Why not just rebuild everything in one engine?

Because each engine earns its place. Docusaurus gives the tech journal proper feeds, reading-time, and code highlighting. The React hub is the portfolio's centerpiece. The field notes are bespoke, hand-designed cheat sheets that would lose their character inside a docs template. The fix wasn't to flatten them into one tool — it was to make them navigable as one collection.

Where to go next

Start at the unified blog index, browse the field notes, or jump straight to one like Spirit in the Sky. Everything is now one hop away.
