Docker Setup to Learn Ansible

Intro​

We usually have to work on different machines, laptops, servers and we lack the shortcuts and all the little things we need to get things done fast. Ansible is a tool that helps us automate and get our main setup up and running quickly. To learn how to use Ansible, I will be using a Docker container so that I can spin up a clean environment and test that my ansible playbooks work as expected. This is the docker setup I am using.

FROM ubuntu:latest

# Allows passing custom build-time arguments. TAGS might be used later to specify Ansible tags.
ARG TAGS

# Sets the working directory inside the container.
#  /usr/local/bin is usually for executables
WORKDIR /usr/local/bin

# Set noninteractive mode for apt to avoid prompting for user input.
ARG DEBIAN_FRONTEND=noninteractive

# Update using apt as package manager, install tools for adding PPAs, add Ansible,
# then install curl, git, ansible, build-essential, and neovim.
RUN apt update && \
    apt install -y software-properties-common && \
    apt-add-repository -y ppa:ansible/ansible && \
    apt update && \
    apt install -y curl git ansible build-essential neovim

# Copy everything from the current directory on the host into the container.
# Placing files into /usr/local/bin
# We are currently copying the entire project directory into the container
# in the future we will move this into public git repo and be able to just pull and run
# we will have folder in that repo that we will encrypt and decrypt with ansible and some will
# be public, the cool thing about having it public is that we can pull without password or ssh key
# this is useful so you can pull from whereever you are or any computer you have ssh access to
# so we will test addding some public files like your ansible local.yml where it doesn't matter who reads it
# we will also store some private files like your ssh key and passwords but encrypt them and only us with password can encyprt them
# so this makes our ansible files private and we can pull from anywhere
COPY . .

# Default command to run when the container starts.
# Runs ansible-playbook with provided TAGS and the local.yml playbook.
# CMD ["sh", "-c", "ansible-playbook $TAGS local.yml"]

# Change the default command to just start a bash shell.
CMD ["bash"]

# TO BUILD THIS IMAGE RUN IN TERMINAL:
#--------------------------------
# docker build -t my-ansible-env .
#--------------------------------
#     Use the current directory (.) as the build context.
#     Build the image defined by Dockerfile.
#     Tag the resulting image as my-ansible-env.



# TO RUN THIS IMAGE RUN IN TERMINAL:
#--------------------------------
# docker run --rm -it my-custom-ansible-env
#--------------------------------
#     --rm removes the container once it exits, preventing “dangling” containers.
#     --it runs the container in interactive mode, allowing you to see the output.
#      my-ansible-env is the name of the image you built.
